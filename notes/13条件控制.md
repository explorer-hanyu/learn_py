# Python3 条件控制


## 一、条件控制概述

Python 条件语句是通过**一条或多条语句的执行结果（True 或者 False）** 来决定执行的代码块。

### 条件判断关键字速查

| 关键字 | 说明 | 示例 |
|--------|------|------|
| `if` | 条件判断语句，当条件为 True 时执行代码块 | `if x > 0:` |
| `elif` | 多条件判断分支（else if） | `elif x == 0:` |
| `else` | 所有条件不满足时执行 | `else:` |
| `pass` | 空语句，占位用，保证语法完整 | `if x > 0: pass` |
| `match` | 结构化模式匹配（Python 3.10+，类似 switch） | `match x: case 1: ...` |




## 二、if 语句

### 2.1 基本语法

Python 中 if 语句的一般形式如下：

```python
if condition_1:
    statement_block_1
elif condition_2:
    statement_block_2
else:
    statement_block_3
```

- 如果 `condition_1` 为 True，将执行 `statement_block_1`
- 如果 `condition_1` 为 False，将判断 `condition_2`
- 如果 `condition_2` 为 True，将执行 `statement_block_2`
- 如果 `condition_2` 为 False，将执行 `statement_block_3`

### 2.2 ⚠️ 关键注意事项

| 规则 | 说明 |
|------|------|
| **冒号 `:`** | 每个条件后面要使用冒号 `:`，表示接下来是满足条件后要执行的语句块 |
| **缩进** | 使用缩进来划分语句块，相同缩进数的语句在一起组成一个语句块 |
| **elif 替代 else if** | Python 中用 `elif` 代替了 `else if`，所以 if 语句的关键字为：`if – elif – else` |
| **无 switch/case** | 在 Python 中没有 `switch...case` 语句，但在 Python 3.10 版本添加了 `match...case` |

### 2.3 简单示例

```python
#!/usr/bin/python3
var1 = 100
if var1:
    print("1 - if 表达式条件为 true")
    print(var1)

var2 = 0
if var2:
    print("2 - if 表达式条件为 true")
    print(var2)
print("Good bye!")
```

**输出结果：**
```
1 - if 表达式条件为 true
100
Good bye!
```

从结果可以看到，由于变量 `var2` 为 0，对应的 if 内的语句没有执行。

> **💡 关键点**：在 Python 中，`0`、`None`、`False`、空字符串 `""`、空列表 `[]`、空元组 `()`、空字典 `{}`、空集合 `set()` 在条件判断中都会被当作 **False**。

### 2.4 实际操作运算符

if 语句中常用的操作运算符：

| 操作符 | 描述 |
|--------|------|
| `<` | 小于 |
| `<=` | 小于或等于 |
| `>` | 大于 |
| `>=` | 大于或等于 |
| `==` | 等于，比较两个值是否相等 |
| `!=` | 不等于 |

**示例：**
```python
# 程序演示了 == 操作符
# 使用数字
print(5 == 6)    # False

# 使用变量
x = 5
y = 8
print(x == y)    # False
```

### 2.5 综合示例：猜数字游戏

```python
#!/usr/bin/python3
# 该实例演示了数字猜谜游戏
number = 7
guess = -1
print("数字猜谜游戏!")
while guess != number:
    guess = int(input("请输入你猜的数字："))
    if guess == number:
        print("恭喜，你猜对了！")
    elif guess < number:
        print("猜的数字小了...")
    elif guess > number:
        print("猜的数字大了...")
```

**输出结果：**
```
数字猜谜游戏!
请输入你猜的数字：1
猜的数字小了...
请输入你猜的数字：9
猜的数字大了...
请输入你猜的数字：7
恭喜，你猜对了！
```




## 三、if 嵌套

在嵌套 if 语句中，可以把 `if...elif...else` 结构放在另外一个 `if...elif...else` 结构中。

### 语法结构

```python
if 表达式1:
    语句
    if 表达式2:
        语句
    elif 表达式3:
        语句
    else:
        语句
elif 表达式4:
    语句
else:
    语句
```

### 示例：判断数字的整除性

```python
#!/usr/bin/python3
num = int(input("输入一个数字："))
if num % 2 == 0:
    if num % 3 == 0:
        print("你输入的数字可以整除 2 和 3")
    else:
        print("你输入的数字可以整除 2，但不能整除 3")
else:
    if num % 3 == 0:
        print("你输入的数字可以整除 3，但不能整除 2")
    else:
        print("你输入的数字不能整除 2 和 3")
```

**输出结果：**
```
输入一个数字：6
你输入的数字可以整除 2 和 3
```




## 四、match...case（Python 3.10+）

Python 3.10 增加了 `match...case` 的条件判断，不需要再使用一连串的 `if-else` 来判断了。

### 4.1 基本语法

```python
match subject:
    case <pattern_1>:
        <action_1>
    case <pattern_2>:
        <action_2>
    case <pattern_3>:
        <action_3>
    case _:
        <action_wildcard>
```

`match` 后的对象会依次与 `case` 后的内容进行匹配，如果匹配成功，则执行匹配到的表达式，否则直接跳过。

- `_` 可以匹配一切，类似于 C 和 Java 中的 `default:`，当其他 case 都无法匹配时，匹配这条，保证永远会匹配成功

### 4.2 示例：HTTP 状态码处理

```python
def http_error(status):
    match status:
        case 400:
            return "Bad request"
        case 404:
            return "Not found"
        case 418:
            return "I'm a teapot"
        case _:
            return "Something's wrong with the internet"

print(http_error(400))   # Bad request
print(http_error(404))   # Not found
print(http_error(418))   # I'm a teapot
print(http_error(500))   # Something's wrong with the internet
```



### 4.3 多条件匹配

一个 `case` 也可以设置多个匹配条件，条件使用 `|` 隔开：

```python
def check_permission(status):
    match status:
        case 200:
            return "OK - 请求成功"
        case 301 | 302:
            return "Redirect - 重定向"
        case 401 | 403 | 404:
            return "Not allowed - 无权限或未找到"
        case 500 | 502 | 503:
            return "Server Error - 服务器错误"
        case _:
            return "Unknown status - 未知状态码"

for code in [200, 301, 403, 500, 418]:
    print(f"状态码 {code}: {check_permission(code)}")
```

**输出结果：**
```
状态码 200: OK - 请求成功
状态码 301: Redirect - 重定向
状态码 403: Not allowed - 无权限或未找到
状态码 500: Server Error - 服务器错误
状态码 418: Unknown status - 未知状态码
```



> **💡 提示**：`match...case` 更多内容参考：[Python match-case 语句](https://www.runoob.com/python3/python-match-case.html)


## 五、重点总结

| 知识点 | 要点 |
|--------|------|
| **if 语句** | 最基本的条件判断，条件为 True 时执行 |
| **elif 子句** | 多个条件分支，按顺序判断 |
| **else 子句** | 所有条件均为 False 时执行（可选） |
| **缩进规则** | 用缩进表示代码块，同一代码块缩进必须一致 |
| **冒号 `:`** | 每个条件判断语句后必须加冒号 |
| **嵌套 if** | if 语句内部可以再嵌套 if 语句 |
| **match...case** | Python 3.10+ 新增，类似 switch/case，支持模式匹配 |
| **布尔上下文** | `0`、`None`、`False`、空序列在条件中视为 False |

## 任务：智能健康建议系统

编写一个名为 `health_advisor.py` 的程序，综合运用条件控制语句完成以下功能。

### 要求

1. **BMI 计算与分类（if / elif / else）**
   - 使用 `input()` 提示用户输入**身高（米）** 和**体重（千克）**
   - 将输入转换为浮点数
   - 计算 BMI = 体重 / (身高 ** 2)
   - 根据 BMI 值输出分类：
     - `BMI < 18.5` → `"体重偏轻"`
     - `18.5 <= BMI < 24` → `"体重正常"`
     - `24 <= BMI < 28` → `"超重"`
     - `BMI >= 28` → `"肥胖"`

2. **运动建议（嵌套 if）**
   - 在上述分类基础上，对**超重**和**肥胖**的用户：
     - 如果 BMI < 26，建议 `"每天快走30分钟"`
     - 如果 BMI >= 26，建议 `"每天慢跑40分钟"`
   - 对**体重正常**的用户：建议 `"保持现有运动习惯"`
   - 对**体重偏轻**的用户：建议 `"增加营养摄入，适当力量训练"`

3. **年龄分组（elif 多分支）**
   - 使用 `input()` 提示用户输入年龄（整数）
   - 根据年龄输出分组：
     - `0-17岁` → `"未成年人"`
     - `18-35岁` → `"青年"`
     - `36-55岁` → `"中年"`
     - `56岁以上` → `"老年"`
   - 如果年龄小于 0，输出 `"年龄输入有误"`

4. **综合健康评估（逻辑组合）**
   - 如果用户 **BMI 正常 且 年龄在 18-35 岁**，输出 `"身体状况优秀，继续保持！"`
   - 如果用户 **（超重或肥胖）且 年龄大于 55 岁**，输出 `"建议定期体检，关注心血管健康"`
   - 其他情况输出 `"建议根据上述建议调整生活习惯"`

5. **match...case 练习（Python 3.10+）**

   在游戏开发中，服务器收到的玩家指令往往以不同的数据结构（元组、列表、字典）传递。请你定义一个函数 `parse_instruction(ins)`，使用 `match...case` 根据传入指令的**结构和内容**返回对应的字符串结果。

**匹配规则如下：**

| 指令格式（传入的参数） | 匹配条件 | 返回值 |
| :--- | :--- | :--- |
| `("move", x, y)` | 元组，第一个元素是 `"move"`，后跟两个数字 | `f"移动至({x}, {y})"` |
| `("move", x, y)` **且** `x > 100` 或 `y > 100` | 在上述元组匹配基础上，增加守卫条件 | `f"长途移动至({x}, {y})"` **（优先于上一条）** |
| `["attack", target]` | 列表，第一个元素是 `"attack"`，后跟一个目标名称 | `f"攻击{target}"` |
| `{"cmd": "talk", "npc": name, "msg": content}` | 字典，包含固定的键值对，并提取 `name` 和 `content` | `f"对{name}说：“{content}”"` |
| `{"cmd": "special", "id": 0}` | 字典，`cmd` 为 `"special"` 且 `id` 恰好为 `0` | `"释放终极技能"` |
| 其他任何情况 | 以上均不匹配 | `"无效指令"` |

6. **边界情况处理**
   - 如果用户输入的体重或身高为负数，输出 `"输入数据无效，请重新运行程序"`
   - 使用 `if` 判断并在程序早期退出（可以使用 `exit()` 或 `return`）
