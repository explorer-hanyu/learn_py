# Python3 解释器
## 一、Python 解释器是什么

Python 程序由**解释器**来执行。解释器将你编写的 Python 代码逐行翻译成计算机能够执行的指令。

> **注意**：教程页面中的 `$` 表示命令提示符，`$` 本身不在输入的命令语句中，无需输入。


## 二、安装与环境变量

### 2.1 Linux / Unix 系统

- 安装后将 Python3 路径添加到环境变量：
  ```bash
  PATH=$PATH:/usr/local/python3/bin/python3
  ```
- 验证安装：
  ```bash
  python3 --version
  ```
  安装完成后，即可通过 shell 终端输入 `python3` 启动解释器。

### 2.2 Windows 系统

- 在 Windows 下通过以下命令设置环境变量（假设 Python 安装在 `C:\Python34`）：
  ```bash
  set path=%path%;C:\python34
  ```


## 三、两种编程模式

Python 支持两种编程模式：**交互式编程** 和 **脚本式编程**。

### 3.1 交互式编程

交互式编程不需要创建脚本文件，直接在 Python 解释器的交互模式中编写代码并**立即获得结果**。

#### 启动交互模式

在终端中输入 `python3` 并回车：

```bash
python3
```

此时将进入交互模式，出现 `>>>` 提示符：

```bash
$ python3
Python 3.4.0 (default, Apr 11 2014, 13:05:11)
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

#### 执行语句

在 `>>>` 后直接输入 Python 语句，按回车即可看到结果：

```python
>>> print ("Hello, Python!")
Hello, Python!
```

#### 多行结构

当键入多行结构（如 `if` 语句）时，续行是必须的，解释器会显示 `...` 提示符：

```python
>>> flag = True
>>> if flag:
...     print("flag 条件为 True!")
...
flag 条件为 True!
```

#### 退出交互模式

- 输入 `exit()` 回车
- 或按 `Ctrl + D`（Linux / Mac）/ `Ctrl + Z`（Windows）

### 3.2 脚本式编程

脚本式编程是将 Python 语句写入后缀为 `.py` 的文件中，然后从外部调用执行。

#### 编写脚本

将以下代码存入 `hello.py` 文件：

```python
print ("Hello, Python!");
```

#### 执行脚本

通过以下命令执行脚本：

```bash
python3 hello.py
```

输出结果：

```
Hello, Python!
```

#### 使脚本可直接执行（Linux/Unix 系统）

在脚本顶部添加 shebang：

```python
#!/usr/bin/env python3
print ("Hello, Python!");
```

修改脚本权限，使其拥有可执行权限：

```bash
chmod +x hello.py
```

然后直接运行：

```bash
./hello.py
```


## 四、启动解释器的多种方式

| 启动方式 | 命令 | 说明 |
|----------|------|------|
| 标准启动 | `python3` | 启动交互式解释器 |
| 执行脚本 | `python3 hello.py` | 执行 `.py` 脚本文件 |
| 命令行执行 | `python3 -c "print('Hello')"` | 执行字符串中的代码 |
| 执行模块 | `python3 -m module_name` | 将模块作为脚本运行 |
| 交互式进入（执行后） | `python3 -i script.py` | 先执行脚本，再进入交互模式 |

> 详细参数说明可通过 `python3 -h` 查看。

### 4.1 交互模式特性：`_` 变量

在 Python 交互模式中，单个下划线 `_` 是一个**特殊变量**，它会**自动保存最后一条表达式求值的结果**：

```python
>>> 3 + 4
7
>>> print(_)      # _ 保存了上一次表达式的结果 7
7
>>> _ * 2
14
```

这个特性在调试和探索性编程中非常实用。但注意 `_` 仅在交互模式下可用，日常脚本代码中不应依赖它。


## 五、多种 Python 解释器介绍

Python 解释器不止一种，以下是几种主要的解释器实现：

| 解释器 | 说明 |
|--------|------|
| **CPython** | C 语言开发的官方标准实现，生态最丰富，应用最广泛 |
| **IPython** | 在 CPython 基础上增强交互体验，支持 tab 补全、历史回溯等功能 |
| **Jython** | 专为 Java 平台设计，将 Python 代码编译成 Java 字节码执行 |
| **PyPy** | 以速度著称，支持 Python 2.7.13 和 3.5.3，通过 JIT（即时编译）加速代码执行 |


## 六、实用技巧汇总

| 技巧 | 说明 |
|------|------|
| 查看 Python 版本 | `python3 --version` |
| 查看帮助 | 在交互模式中键入 `help()` |
| 使用 `_` 变量 | 交互模式中 `_` 保存上一次表达式结果 |
| `;` 分隔语句 | 交互模式中可用 `;` 在一行写入多个简单语句 |
| 多行续行 | 交互模式中，多行结构会自动进入续行模式（`...` 提示符） |
| 命令行执行片段 | `python3 -c "print('test')"` — 无需文件即可快速测试代码 |
