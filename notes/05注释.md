# Python3 注释笔记

## 一、注释的作用

在 Python 中，注释**不会影响程序的执行**，但会使代码更易于阅读和理解。

## 二、单行注释

### 语法
Python 中单行注释以 `#` 开头，`#` 符号后面的**所有文本都被视为注释**，不会被解释器执行。

### 示例

```python
# 这是一个注释
print("Hello, World!")  # 这也是注释
```

### 行尾注释建议
在代码结尾处添加注释时，建议在 `#` 前留两个空格，以提高可读性。

```python
total = price * quantity  # 计算总价
```

## 三、多行注释

Python 中没有专门的多行注释语法，但可以使用**多行字符串**（由三个单引号 `'''` 或三个双引号 `"""` 包围的文本块）来实现多行注释的效果。

### 1. 使用三个单引号

```python
#!/usr/bin/python3

'''
这是多行注释，用三个单引号
这是多行注释，用三个单引号
这是多行注释，用三个单引号
'''
print("Hello, World!")
```

### 2. 使用三个双引号

```python
#!/usr/bin/python3

"""
这是多行注释，用三个双引号
这是多行注释，用三个双引号
这是多行注释，用三个双引号
"""
print("Hello, World!")
```

### ⚠️ 注意事项

| 要点 | 说明 |
|------|------|
| **实质是字符串** | 多行字符串在这里被当作多行注释使用，但它实际上是一个字符串。只要不使用它，它不会影响程序的运行。 |
| **不能嵌套使用** | 多行注释不能嵌套。当你开始一个多行注释块时，Python 会一直将后续行都当作注释，直到遇到另一组三个引号。嵌套会导致语法错误。 |

### ❌ 错误示例（嵌套多行注释）

```python
'''
这是外部的多行注释
可以包含一些描述性的内容
'''  # 这里已经结束了外部注释
这是尝试嵌套的多行注释  # 这行不再是注释，会导致语法错误
'''
```

### ✅ 正确做法（使用单行注释嵌套）

```python
'''
这是外部的多行注释
可以包含一些描述性的内容
# 这是内部的单行注释
# 可以嵌套在多行注释中
'''
```

## 四、Docstring 文档字符串

### 概述

Python 的 **Docstring（文档字符串）** 是一种特殊的注释，用于为函数、类、模块等添加文档说明。它类似于 Java 的 Javadoc，但更加强大和灵活。

**Docstring 与普通注释的关键区别：**
- Docstring 可以通过 `__doc__` 属性直接访问
- 可以使用 `help()` 函数查看
- 普通注释则无法被程序读取

### 基本语法

Docstring 使用三个双引号 `"""` 或三个单引号 `'''` 包围，**放在函数、类、模块的开头**。

### 函数 Docstring 示例

```python
def add(a, b):
    """返回两数之和"""
    return a + b

# 通过 __doc__ 属性访问
print(add.__doc__)  # 输出: 返回两数之和

# 使用 help() 函数查看
help(add)
```
运行 `help(add)` 预期输出：
```
Help on function add in module __main__:
add(a, b)
    返回两数之和
```

### 使用 inspect 模块提取文档

Python 标准库提供了 `inspect` 模块，可以直接提取文档内容：

```python
import inspect

def add(a, b):
    """返回两数之和"""
    return a + b

# 使用 inspect.getdoc() 获取文档
print(inspect.getdoc(add))  # 输出: 返回两数之和
```

### 多行 Docstring

对于复杂的函数，可以使用多行 Docstring：

```python
def calculate(a, b, operation="add"):
    """
    执行数学运算

    参数:
        a: 第一个数字
        b: 第二个数字
        operation: 操作类型，可选 "add", "subtract", "multiply"

    返回:
        计算结果
    """
    if operation == "add":
        return a + b
    elif operation == "subtract":
        return a - b
    elif operation == "multiply":
        return a * b
    else:
        raise ValueError("不支持的操作")
```

### 类的 Docstring

Docstring 也可以用于类：

```python
class Person:
    """人物类，用于表示一个人的基本信息"""

    def __init__(self, name, age):
        """
        初始化人物对象

        参数:
            name: 姓名
            age: 年龄
        """
        self.name = name
        self.age = age

    def introduce(self):
        """介绍这个人"""
        return f"我叫{self.name}，今年{self.age}岁"

# 访问类的文档
print(Person.__doc__)  # 输出: 人物类，用于表示一个人的基本信息
```

## 五、Docstring 风格规范

Python 中有多种 Docstring 风格，常用的有：

| 风格 | 特点 |
|------|------|
| **Google 风格** | 使用空格缩进，参数和返回值有明确的标签 |
| **Sphinx/reST 风格** | 使用冒号开头，如 `:param name:` |
| **NumPy 风格** | 与 Google 风格类似，但格式略有不同 |

> **建议**：在项目中选择一种风格并保持一致。

## 六、注释最佳实践

### 原则

| 原则 | 说明 |
|------|------|
| **解释"为什么"而非"做什么"** | 重点说明代码的意图或特殊处理原因 |
| **避免冗余注释** | 代码逻辑清晰时无需重复说明 |
| **保持缩进一致** | 每一层级的代码注释应当与那一层级的代码缩进保持一致 |
| **注释不是越多越好** | 对于一目了然的代码，不需要添加注释 |

### 正确示例

```python
# 防止设备过热损坏（说明原因，而非描述代码做了什么）
if temperature > 100:
    shutdown()
```

### 注释与 Docstring 对比

| 对比项 | 普通注释（#） | Docstring |
|--------|---------------|-----------|
| 访问方式 | 不可被程序读取 | 可通过 `__doc__` 属性访问 |
| 使用位置 | 任意位置 | 函数、类、模块的开头 |
| 主要用途 | 辅助理解代码 | 提供正式的 API 文档 |
| 工具支持 | 无 | `help()`、`inspect` 模块可提取 |

## 任务：员工管理系统注释完善

你拿到了一份**员工信息管理**的代码，但原开发者没有添加任何注释，你需要完成以下**所有要求**，确保每个知识点都有体现：

```python
class Employee:
    def __init__(self, emp_id, name, salary):
        self.emp_id = emp_id
        self.name = name
        self.salary = salary

    def annual_income(self):
        return self.salary * 12

    def give_raise(self, percent):
        self.salary = self.salary * (1 + percent / 100)

def get_employee_info(emp):
    info = f"员工ID: {emp.emp_id}\n姓名: {emp.name}\n月薪: {emp.salary}"
    return info

emp1 = Employee("E001", "张三", 8000)
emp2 = Employee("E002", "李四", 9500)
```

### 具体要求

1. **单行注释**：在代码顶部添加文档说明，注明文件名、作者和日期（使用 `#` 注释）。

2. **多行注释**：
   - 使用 `'''` 或 `"""` 形式，在 `give_raise` 方法前添加一段**多行注释**，说明该方法的功能和使用注意事项（特别是 `percent` 参数的含义——是百分点而非倍数，例如传入 `10` 表示加薪 10%）。
   - 在 `get_employee_info` 函数前使用**另一种引号形式**添加多行注释。

3. **Docstring**：
   - 为 `Employee` 类添加 Docstring，描述这个类的用途和属性。
   - 为 `__init__` 方法添加 Docstring，说明参数含义。
   - 为 `annual_income` 方法添加 Docstring，描述返回值含义。
   - 使用 `print(Employee.__doc__)` 输出类的文档。
   - 使用 `help(Employee.annual_income)` 查看方法的帮助文档。

4. **行尾注释**：
   - 在 `emp2` 的赋值语句后面添加行尾注释，标注李四的岗位（如 `# 技术主管`）。
   - 在 `annual_income` 方法的 `return` 行添加行尾注释，说明计算逻辑。

5. **注释最佳实践**：
   - 在代码中找一个你认为逻辑不够清晰的地方，添加一行**解释原因而非描述行为**的注释。
   - 添加一个 `# TODO:` 标记注释，提醒后续需要增加的功能（如"添加年终奖计算"）。
